# 1.2 线程模型 - 速查卡

> **本章聚焦**：线程同步机制（ENQUEUE原理、Lambda捕获）、Fence、TaskGraph、无锁队列
> 
> **渲染调用链**（FEngineLoop → FSceneRenderer）→ 详见 [1.1 RenderLoop](./1.1_RenderLoop.md)

## 🎯 核心结论

1. **三线程架构**：GameThread → RenderThread → RHIThread
2. **命令投递**：`ENQUEUE_RENDER_COMMAND` 将GT任务发送到RT（**必须按值捕获**）
3. **同步Fence**：`FRenderCommandFence` / `FlushRenderingCommands()` 实现GT等待RT
4. **Task Graph**：任务依赖图，自动调度到Worker线程并行执行
5. **无锁队列**：GT→RT通信使用SPSC无锁队列，避免锁竞争

## 📊 线程职责

| 线程 | 职责 | 典型耗时 |
|------|------|---------|
| GameThread | 游戏逻辑、物理、AI、输入 | 5-8ms |
| RenderThread | 可见性剔除、命令生成、资源更新 | 5-10ms |
| RHIThread | 翻译为D3D12/Vulkan、GPU提交 | 2-5ms |

## 📝 ENQUEUE_RENDER_COMMAND

```cpp
// ✅ 正确：值捕获
float Time = 1.0f;
ENQUEUE_RENDER_COMMAND(MyCmd)(
    [Time](FRHICommandListImmediate& RHICmdList)
    {
        // Time是副本，安全
    });

// ❌ 危险：引用捕获
ENQUEUE_RENDER_COMMAND(BadCmd)(
    [&Time](FRHICommandListImmediate& RHICmdList)  // 危险！
    {
        // Time可能已销毁
    });

// ✅ 大对象：移动语义
TArray<FVector> Data = ...;
ENQUEUE_RENDER_COMMAND(MoveCmd)(
    [Data = MoveTemp(Data)](FRHICommandListImmediate& RHICmdList)
    {
        // Data被移动，不是拷贝
    });

// ✅ UObject：弱引用
TWeakObjectPtr<AActor> WeakActor = MyActor;
ENQUEUE_RENDER_COMMAND(SafeCmd)(
    [WeakActor](FRHICommandListImmediate& RHICmdList)
    {
        if (AActor* Actor = WeakActor.Get()) { ... }
    });
```

## 🔗 同步机制

```cpp
// 等待所有渲染命令完成
FlushRenderingCommands();

// 精细控制
FRenderCommandFence Fence;
Fence.BeginFence();           // 插入Fence
Fence.Wait();                 // 阻塞等待
bool bDone = Fence.IsFenceComplete();  // 非阻塞检查
```

## 📊 Task Graph

```cpp
// 简单并行循环
ParallelFor(1000, [](int32 Index) {
    ProcessItem(Index);
});

// 异步任务
TFuture<int32> Future = Async(EAsyncExecution::ThreadPool, []() {
    return ComputeSomething();
});
int32 Result = Future.Get();  // 等待结果

// 带依赖的任务
FGraphEventRef TaskA = FFunctionGraphTask::CreateAndDispatchWhenReady(
    []() { LoadData(); }, TStatId());

FGraphEventRef TaskB = FFunctionGraphTask::CreateAndDispatchWhenReady(
    []() { ProcessData(); }, TStatId(), TaskA);  // B依赖A
```

## 🔑 线程检查函数

```cpp
IsInGameThread();        // 是否在GT
IsInRenderingThread();   // 是否在RT
IsInRHIThread();         // 是否在RHIT

check(IsInGameThread()); // 断言
```

## 📊 性能对比

| 方案 | 入队耗时 | 适用场景 |
|------|---------|---------|
| 锁队列 | ~100ns | 低频访问 |
| SPSC无锁 | ~20ns | 单生产者单消费者（GT→RT） |
| MPSC无锁 | ~30ns | 多生产者单消费者 |
