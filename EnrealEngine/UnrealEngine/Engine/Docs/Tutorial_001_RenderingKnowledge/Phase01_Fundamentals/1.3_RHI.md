# 1.3 RHI抽象层 - 速查卡

## 🎯 核心结论

1. **RHI存在意义**：让渲染代码只写一次，跨平台运行（D3D12/Vulkan/Metal）
2. **核心抽象**：FRHIResource（纹理/缓冲区）、FRHICommandList（命令）、FRHIPipelineState（PSO）
3. **三层架构**：Renderer → RHI接口 → 平台实现（D3D12RHI/VulkanRHI）
4. **Command系统**：延迟执行 + 命令队列，支持多线程录制
5. **类型转换**：UE类型（PF_R8G8B8A8）↔ 平台类型（DXGI_FORMAT_R8G8B8A8_UNORM）

## 📊 架构图

```
┌─────────────────────────────────────────────┐
│          上层渲染代码（RenderGraph）          │
├─────────────────────────────────────────────┤
│              FRHICommandList                │  ← 统一接口
│     RHICreateTexture / RHICmdList->Draw     │
├──────────────┬──────────────┬───────────────┤
│  D3D12RHI    │   VulkanRHI  │   MetalRHI    │  ← 平台实现
├──────────────┼──────────────┼───────────────┤
│  DirectX 12  │    Vulkan    │    Metal      │  ← 原生API
└──────────────┴──────────────┴───────────────┘
```

## 📝 常用API

```cpp
// 创建纹理
FRHITextureCreateDesc Desc = FRHITextureCreateDesc::Create2D(TEXT("MyTex"))
    .SetExtent(1024, 1024)
    .SetFormat(PF_R8G8B8A8);
FRHITexture* Tex = RHICreateTexture(Desc);

// 创建缓冲区
FRHIResourceCreateInfo CreateInfo(TEXT("MyBuffer"));
FBufferRHIRef Buffer = RHICmdList.CreateBuffer(Size, Usage, Stride, Access, CreateInfo);

// 设置RenderTarget
FRHIRenderPassInfo RPInfo(ColorRT, ERenderTargetActions::Clear_Store);
RHICmdList.BeginRenderPass(RPInfo, TEXT("MyPass"));
```

## 🆚 Unity对比

| 维度 | UE5 RHI | Unity |
|------|---------|-------|
| 抽象层 | FDynamicRHI（多态） | CommandBuffer |
| 创建资源 | RHICreateTexture() | new Texture2D() |
| 命令录制 | FRHICommandList | CommandBuffer |
| PSO管理 | 显式（FGraphicsPipelineState） | 隐式 |
| 线程模型 | RHIThread独立提交 | SRP/CommandBuffer 公开层提交，底层线程与后端细节由 Unity 隐藏 |
