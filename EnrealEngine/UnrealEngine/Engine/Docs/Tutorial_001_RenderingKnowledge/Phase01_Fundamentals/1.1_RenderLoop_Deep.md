# 第一部分：理解渲染循环的存在意义

## 1.1 从一个实际问题开始

假设你是引擎开发者，需要设计一个每秒渲染60帧的游戏引擎。

### 1.1.1 最简单的渲染循环

```cpp
// ❌ 最简单但效率最低的方案
while (bRunning)
{
    // 1. 更新游戏逻辑
    UpdateGameLogic();      // 移动角色、计算AI、处理输入...
    
    // 2. 渲染
    RenderScene();          // 遍历场景、提交DrawCall...
    
    // 3. 等待GPU完成
    WaitForGPU();           // 阻塞！
    
    // 4. 显示到屏幕
    Present();
}
```

**问题分析**：

```
时间轴 (单帧 16.7ms @ 60fps)
─────────────────────────────────────────────────────────────
│ CPU: UpdateGame │ CPU: Render │ ← CPU等待 → │ GPU: 执行 │
│      5ms        │    3ms      │    8ms      │    8ms    │
─────────────────────────────────────────────────────────────
                                ↑
                           CPU在这里空等GPU！
```

**核心问题**：CPU和GPU串行工作，互相等待

| 资源 | 忙碌时间 | 空闲时间 | 利用率 |
|------|---------|---------|--------|
| CPU | 8ms | 8ms | 50% |
| GPU | 8ms | 8ms | 50% |
| **总帧时间** | **16ms** | - | - |

### 1.1.2 问题的本质

**CPU和GPU是两个独立的处理器**，它们有各自的工作队列：

```
┌─────────────────────────────────────────────────────────────────┐
│                        硬件架构                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────┐                     ┌─────────────┐          │
│   │    CPU      │                     │     GPU     │          │
│   │  (多核心)    │                     │  (数千核心)  │          │
│   │             │                     │             │          │
│   │ 擅长：      │                     │ 擅长：       │          │
│   │ • 复杂逻辑   │                     │ • 并行计算   │          │
│   │ • 分支判断   │ ← PCIe总线连接 →    │ • 矩阵运算   │          │
│   │ • 低延迟    │                     │ • 大吞吐量   │          │
│   └─────────────┘                     └─────────────┘          │
│                                                                 │
│   【关键】：CPU发送命令，GPU执行命令，通过命令队列异步通信        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

如果CPU必须等GPU完成才能继续，就浪费了CPU的处理能力。

---

## 1.2 UE的解决方案：三线程流水线

### 1.2.1 核心思想

**让CPU和GPU同时工作，处理不同帧的数据**：

```
帧 N:    CPU处理 → GPU执行
帧 N+1:           CPU处理 → GPU执行
帧 N+2:                    CPU处理 → GPU执行

时间轴：
─────────────────────────────────────────────────────────────────
帧 N   │ CPU: Game+Render │                                      │
       │       8ms        │                                      │
─────────────────────────────────────────────────────────────────
帧 N   │                  │ GPU: 执行帧N的命令                    │
       │                  │          8ms                         │
─────────────────────────────────────────────────────────────────
帧 N+1 │                  │ CPU: Game+Render │                   │
       │                  │       8ms        │                   │
─────────────────────────────────────────────────────────────────
                          ↑
                     CPU和GPU同时工作！
```

**效果**：
- 总帧时间 = max(CPU时间, GPU时间)
- 如果 CPU 8ms + GPU 8ms，帧时间可以降到 ~8ms（而不是 16ms）
- **吞吐量翻倍**

### 1.2.2 UE5的三线程架构

UE将CPU工作进一步拆分到三个线程：

```
┌─────────────────────────────────────────────────────────────────┐
│                    UE5的三线程架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   【GameThread】              【RenderThread】      【RHIThread】 │
│    游戏逻辑线程                 渲染命令生成           GPU命令提交 │
│                                                                 │
│   ┌─────────────┐          ┌─────────────┐     ┌─────────────┐ │
│   │ • Tick逻辑  │          │ • 可见性剔除 │     │ • 翻译为    │ │
│   │ • 物理模拟  │ ───────> │ • 构建命令  │ ──> │   D3D12/    │ │
│   │ • AI计算    │ 任务队列  │ • 排序批次  │命令  │   Vulkan    │ │
│   │ • 输入处理  │          │ • 资源更新  │队列  │ • GPU提交   │ │
│   └─────────────┘          └─────────────┘     └─────────────┘ │
│         │                         │                   │        │
│         │                         │                   │        │
│     Frame N              Frame N-1              Frame N-2      │
│     (最新)               (1帧延迟)              (2帧延迟)       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 线程 | 职责 | 时间消耗 |
|------|------|----------|
| **GameThread** | 游戏逻辑、物理、AI、输入 | ~5-8ms |
| **RenderThread** | 可见性计算、命令生成、资源更新 | ~5-10ms |
| **RHIThread** | 翻译命令为平台API、提交GPU | ~2-5ms |

> 💡 **线程模型详解**：三线程的创建、同步、Task Graph等机制详见 [1.2 线程模型](./1.2_ThreadModel_Deep.md)

## 1.3 本章聚焦：渲染调用链

本章重点讲解**从函数调用角度**理解渲染循环：

```
【GameThread】                     【RenderThread】                 【RHIThread】
    │                                    │                              │
    │ FEngineLoop::Tick()               │                              │
    │  ├─ 更新时间                       │                              │
    │  ├─ Tick游戏逻辑                   │                              │
    │  │   └─ AActor::Tick()            │                              │
    │  │   └─ UWorld::Tick()            │                              │
    │  │                                 │                              │
    │  └─ 触发渲染                       │                              │
    │      └─ UGameEngine::Tick()       │                              │
    │          └─ Viewport->Draw()      │                              │
    │              │                     │                              │
    │              └──ENQUEUE───────────>FSceneRenderer::Render()     │
    │                                    │  ├─ InitViews()             │
    │                                    │  ├─ RenderPrePass()         │
    │                                    │  ├─ RenderBasePass()        │
    │                                    │  ├─ RenderLighting()        │
    │                                    │  └─ PostProcessing()        │
    │                                    │      │                       │
    │                                    │      └──Submit──────────────>│
    │                                    │                              │
    │                                    │                              │ D3D12/Vulkan
    │                                    │                              │ GPU提交
    │                                    │                              │
   Frame N                           Frame N-1                     Frame N-2
```

💡 **核心要点**
- UE渲染循环的核心是 `FEngineLoop::Tick()` → `FSceneRenderer::Render()`
- GameThread触发渲染，RenderThread执行渲染，RHIThread提交GPU
- 本章聚焦调用链和源码，线程同步机制详见1.2

## 2.1 FEngineLoop::Tick() - 引擎主循环

### 2.1.1 函数概览

**来源**：`Engine/Source/Runtime/Launch/Private/LaunchEngineLoop.cpp`  
**源码锚点**：搜索 `FEngineLoop::Tick`、`GEngine->Tick`、`FFrameEndSync::Sync`。  
**职责**：引擎每帧的入口点，协调所有子系统

### 2.1.2 核心流程

```cpp
// ✅ 源码摘要 (UE5.7)
// 来源：LaunchEngineLoop.cpp

void FEngineLoop::Tick()
{
    // 【步骤1：性能追踪】
    SCOPED_NAMED_EVENT_TEXT("FEngineLoop::Tick", FColor::Red);
    
    // 【步骤2：心跳检测】
    // 用于检测主线程是否卡死
    FThreadHeartBeat::Get().HeartBeat();
    
    // 【步骤3：更新时间】
    // 计算DeltaTime，处理帧率限制
    GEngine->UpdateTimeAndHandleMaxTickRate();
    
    // 【步骤4：核心 - 调用引擎Tick】
    // 这里会触发所有游戏逻辑和渲染
    GEngine->Tick(FApp::GetDeltaTime(), bIdleMode);
    
    // 【步骤5：Tick渲染相关】
    // 如果是单线程模式，需要手动Tick渲染
    if (!GUseThreadedRendering)
    {
        TickRenderingTickables(FRHICommandListImmediate::Get());
    }
    
    // 【步骤6：帧结束同步】
    // 确保各线程同步
    FFrameEndSync::Sync();
    
    // 【步骤7：帧计数+1】
    ++GFrameCounter;
}
```

## 2.2 UGameEngine::Tick() - 游戏引擎Tick

### 2.2.1 函数概览

**来源**：`Engine/Source/Runtime/Engine/Private/GameEngine.cpp`  
**源码锚点**：搜索 `UGameEngine::Tick`、`RedrawViewports`、`GameViewport->Viewport->Draw`。  
**职责**：Tick所有World的游戏逻辑，触发渲染

### 2.2.2 核心流程

```cpp
// ✅ 源码摘要 (UE5.7)
// 来源：GameEngine.cpp

void UGameEngine::Tick(float DeltaSeconds, bool bIdleMode)
{
    // 【步骤1：参数验证】
    if (DeltaSeconds < 0.0f)
    {
        UE_LOG(LogEngine, Fatal, TEXT("Negative delta time!"));
    }
    
    // 【步骤2：Tick所有World】
    for (FWorldContext& Context : WorldList)
    {
        if (UWorld* World = Context.World())
        {
            // 这里会调用所有Actor的Tick
            World->Tick(LEVELTICK_All, DeltaSeconds);
        }
    }
    
    // 【步骤3：触发渲染】
    if (GameViewport && GameViewport->Viewport)
    {
        // 关键调用！
        RedrawViewports(true);
    }
    
    // 【步骤4：其他系统Tick】
    // - Audio
    // - Physics
    // - Network
    // - GC检查
}
```

### 2.2.3 RedrawViewports详解

```cpp
// 源码锚点：Engine/Source/Runtime/Engine/Private/GameEngine.cpp，搜索 UGameEngine::RedrawViewports

void UGameEngine::RedrawViewports(bool bShouldPresent)
{
    // 【步骤1：布局玩家视口】
    // 分屏模式下计算每个玩家的屏幕区域
    GameViewport->LayoutPlayers();
    
    // 【步骤2：锁定Slate资源】
    // 防止与LoadingScreen线程冲突
    FSlateRenderer* SlateRenderer = FSlateApplication::Get().GetRenderer();
    FScopeLock ScopeLock(SlateRenderer->GetResourceCriticalSection());
    
    // 【步骤3：触发Draw】
    GameViewport->Viewport->Draw(bShouldPresent);
}
```

💡 **核心要点**：`UGameEngine::Tick()` 先Tick游戏逻辑，再通过 `RedrawViewports` 触发渲染

## 2.3 FViewport::Draw() - 触发渲染

### 2.3.1 函数概览

**来源**：`Engine/Source/Runtime/Engine/Private/UnrealClient.cpp`  
**源码锚点**：搜索 `FViewport::Draw`、`EnqueueBeginRenderFrame`、`EnqueueEndRenderFrame`。  
**职责**：协调渲染流程，发送Begin/End Frame命令

### 2.3.2 核心流程

```cpp
// ✅ 源码摘要 (UE5.7)
// 来源：UnrealClient.cpp

void FViewport::Draw(bool bShouldPresent)
{
    SCOPED_NAMED_EVENT(FViewport_Draw, FColor::Red);
    
    // 【防重入检查】
    static bool bReentrant = false;
    if (bReentrant) return;
    bReentrant = true;
    
    // 【步骤1：发送BeginFrame到RenderThread】
    EnqueueBeginRenderFrame(bShouldPresent);
    
    // 【步骤2：在GameThread准备Canvas并绘制】
    FCanvas Canvas(this, nullptr, World, ...);
    
    // 【步骤3：调用ViewportClient::Draw】
    // 这里会构建ViewFamily，触发场景渲染
    ViewportClient->Draw(this, &Canvas);
    
    // 【步骤4：Flush Canvas】
    Canvas.Flush_GameThread();
    
    // 【步骤5：发送EndFrame到RenderThread】
    EnqueueEndRenderFrame(bLockToVsync, bShouldPresent);
    
    bReentrant = false;
}
```

### 2.3.3 EnqueueBeginRenderFrame实现

```cpp
// 源码锚点：Engine/Source/Runtime/Engine/Private/UnrealClient.cpp，搜索 FViewport::EnqueueBeginRenderFrame

void FViewport::EnqueueBeginRenderFrame(const bool bShouldPresent)
{
    FViewport* Viewport = this;
    
    // 关键：将BeginFrame任务发送到RenderThread
    ENQUEUE_RENDER_COMMAND(BeginDrawingCommand)(
        [Viewport](FRHICommandListImmediate& RHICmdList)
        {
            Viewport->BeginRenderFrame(RHICmdList);
        }
    );
}
```

> 💡 **ENQUEUE_RENDER_COMMAND详解**：这个宏的工作原理、Lambda捕获机制详见 [1.2 线程模型](./1.2_ThreadModel_Deep.md)

### 2.3.4 ViewportClient::Draw - 构建ViewFamily

```cpp
// 源码锚点：Engine/Source/Runtime/Engine/Private/GameViewportClient.cpp，搜索 UGameViewportClient::Draw

void UGameViewportClient::Draw(FViewport* InViewport, FCanvas* SceneCanvas)
{
    // 【步骤1：创建ViewFamily】
    FSceneViewFamilyContext ViewFamily(FSceneViewFamily::ConstructionValues(
        InViewport,
        GetWorld()->Scene,
        EngineShowFlags
    ));
    
    // 【步骤2：为每个本地玩家创建View】
    for (FLocalPlayerIterator It(GEngine, GetWorld()); It; ++It)
    {
        FSceneView* View = CalcSceneView(&ViewFamily, ...);
        ViewFamily.Views.Add(View);
    }
    
    // 【步骤3：触发渲染！】
    // 这里将ViewFamily发送给渲染器
    GetRendererModule().BeginRenderingViewFamily(SceneCanvas, &ViewFamily);
}
```

### 2.3.5 核心类关系

```
FViewport (视口)
    │
    ├──> FViewportClient (抽象接口)
    │       │
    │       └──> UGameViewportClient (游戏视口实现)
    │               │
    │               └──> Draw() ───> BeginRenderingViewFamily()
    │
    └──> FSceneViewFamily (视图集合)
            │
            └──> FSceneView (单个视图)
                    ├─ ViewMatrix (视图矩阵)

```

💡 **核心要点**：`FViewport::Draw()` 包装了 Begin/End Frame，核心是通过 `ViewportClient->Draw()` 构建 ViewFamily 并触发渲染

## 2.4 FSceneRenderer::Render() - 核心渲染

### 2.4.1 函数概览

**来源**：`Engine/Source/Runtime/Renderer/Private/DeferredShadingRenderer.cpp`  
**源码锚点**：搜索 `FDeferredShadingSceneRenderer::Render`、`RenderPrePass`、`RenderBasePass`、`RenderLights`。  
**职责**：延迟渲染管线的核心入口

### 2.4.2 核心流程（简化）

```cpp
// ✅ 源码摘要 (UE5.7) - 延迟渲染流程
// 来源：DeferredShadingRenderer.cpp

void FDeferredShadingSceneRenderer::Render(FRDGBuilder& GraphBuilder)
{
    // 【阶段1：初始化和可见性】
    InitViews(GraphBuilder);           // 计算可见性，剔除
    
    // 【阶段2：PrePass】
    RenderPrePass(GraphBuilder);       // 深度预渲染
    
    // 【阶段3：BasePass】
    RenderBasePass(GraphBuilder);      // GBuffer填充
    
    // 【阶段4：光照】
    RenderLighting(GraphBuilder);      // 延迟光照计算
    
    // 【阶段5：透明物体】
    RenderTranslucency(GraphBuilder);  // 透明物体渲染
```

```cpp
void FSceneRenderer::InitViews(FRDGBuilder& GraphBuilder)
{
    // 【1】视锥剔除
    // 使用八叉树或BVH快速排除视锥外的物体
    ComputeViewVisibility(GraphBuilder);
    
    // 【2】遮挡剔除（可选）
    // 使用HZB（Hierarchical Z-Buffer）或软件光栅化
    if (bUseOcclusionCulling)
    {
        OcclusionCull(GraphBuilder);
    }
    
    // 【3】收集可见Primitive
    // 构建FMeshBatch数组
    GatherDynamicMeshElements();
    
    // 【4】排序
    // 按材质、状态等排序以减少状态切换
    SortPrimitives();
}
```

### 2.4.5 RenderBasePass - GBuffer填充

```cpp
// 来源：BasePassRendering.cpp

void FDeferredShadingSceneRenderer::RenderBasePass(FRDGBuilder& GraphBuilder)
{
    // 【1】设置RenderTarget
    // GBufferA/B/C/D + SceneDepth
    FRDGTextureRef GBufferA = GraphBuilder.CreateTexture(...);
    FRDGTextureRef GBufferB = GraphBuilder.CreateTexture(...);
    
    // 【2】添加渲染Pass
    AddPass(GraphBuilder, RDG_EVENT_NAME("BasePass"),
        [&](FRHICommandList& RHICmdList)
        {
            // 【3】遍历所有可见Mesh
            for (const FMeshBatch& Mesh : VisibleMeshes)
            {
                // 【4】设置PSO（Pipeline State Object）
                SetGraphicsPipelineState(RHICmdList, Mesh.Material.PSO);
                
                // 【5】设置Shader参数
                SetShaderParameters(RHICmdList, Mesh);
                
                // 【6】绘制
                RHICmdList.DrawIndexedPrimitive(
                    Mesh.IndexBuffer,
                    Mesh.BaseVertexIndex,
                    Mesh.NumVertices,
                    Mesh.FirstIndex,
                    Mesh.NumPrimitives,
                    Mesh.NumInstances
                );
            }
        }
    );
}
```

### 2.4.6 RenderLighting - 光照计算

```cpp
// 来源：LightRendering.cpp

void FDeferredShadingSceneRenderer::RenderLighting(FRDGBuilder& GraphBuilder)
{
    // 【1】直接光照
    RenderDirectLighting(GraphBuilder);
    
    // 【2】间接光照（Lumen或SSGI）
    if (bUseLumen)
    {
        RenderLumenGlobalIllumination(GraphBuilder);
    }
    else
    {
        RenderSSGI(GraphBuilder);
    }
    
    // 【3】环境光照
    RenderAmbientOcclusion(GraphBuilder);
    
    // 【4】合成
    ComposeLighting(GraphBuilder);
}
```

💡 **核心要点**：`FSceneRenderer::Render()` 是渲染管线的心脏，按顺序执行各个渲染阶段

## 3.3 性能分析

### 3.3.1 stat unit 解读

```
Frame:  16.67 ms   ← 总帧时间
Game:    8.32 ms   ← GameThread时间
Draw:    7.51 ms   ← RenderThread时间
GPU:     5.82 ms   ← GPU时间
RHIT:    2.13 ms   ← RHIThread时间
```

**瓶颈判断**：
- Frame ≈ Game → GameThread瓶颈（游戏逻辑太重）
- Frame ≈ Draw → RenderThread瓶颈（DrawCall太多）
- Frame ≈ GPU → GPU瓶颈（着色器太复杂/像素太多）

> 💡 **瓶颈分析详解**：更详细的瓶颈分析和优化方案见 [1.2 线程模型](./1.2_ThreadModel_Deep.md)
