# 3.1 Nanite虚拟几何系统 - 速查卡

## 🎯 核心结论

1. **虚拟几何**：自动LOD（像素级连续），支持亿级三角形，无Pop-in
2. **Cluster**：~128三角形为一组，GPU剔除最小调度单元
3. **GPU-Driven**：五阶段全在GPU执行（Compute Shader），DrawCall<20
4. **混合光栅化**：大三角形硬件光栅化 + 小三角形软件光栅化（并行重叠）
5. **Visibility Buffer**：64-bit/像素，延迟Material Shading零Overdraw

## 📊 五阶段管线

```
Instance Culling (Compute)
  ↓ Frustum + Distance + WPO
Cluster Culling + LOD Selection (Compute)
  ↓ Frustum + 背面 + HZB + LOD误差
Rasterization (Graphics + Async Compute 并行)
  ├─ HW Rasterize (大三角形)
  └─ SW Rasterize (小三角形)
  ↓ Visibility Buffer (64-bit)
Material Classify (Compute)
  ↓ 按MaterialID分组像素
Material Shading (Compute × N材质)
  ↓ GBuffer → Lighting Pass
```

## 📊 性能估算（RTX 4090, 1080p）

| 阶段 | 估算耗时 | 说明 |
|------|---------|------|
| Instance Culling | ~0.5ms | 100万Instance |
| Cluster Culling | ~0.8ms | 数百万Cluster |
| Rasterization | ~1.8ms | 5000万三角形（HW+SW并行） |
| Material Classify | ~0.3ms | - |
| Material Shading | ~2.8ms | 50种材质 |
| **总计** | **~6-7ms** | 取决于场景复杂度 |

## 🆚 Unity对比

| 维度 | UE5 Nanite | Unity 传统 MeshRenderer / LODGroup 路径 |
|------|------------|----------|
| LOD | 自动像素级 | 手动5-7级 |
| 剔除 | GPU Cluster级 | CPU Mesh级 |
| 光栅化 | 软+硬混合 | 纯硬件 |
| DrawCall | <20 | 数千~数万 |
| 运行时内存 | ~300MB（流式） | ~1.2GB（全LOD） |
