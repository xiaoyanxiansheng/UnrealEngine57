# 3.4 MegaLights - 速查卡

## 🎯 核心结论

1. **随机采样**：每像素只采样2-4个光源，不遍历所有光源 → 开销与光源总数近乎无关
2. **双阴影路径**：Hardware RT（DXR）或 Virtual Shadow Maps（VSM），所有光源可投射阴影
3. **时空降噪**：Spatial Filter（保边去噪）+ Temporal Accumulation（多帧混合） + Visible Light Hash（ghost检测）
4. **Tile分类**：按材质复杂度分类（Simple/Complex/Rect/Textured），不同Tile用不同着色复杂度
5. **历史引导**：上一帧可见性信息引导当前帧采样，减少无效采样

## 📊 MegaLights管线

```
GenerateMegaLightsSamples()（在VSM标记之前）
├─ Setup → 初始化参数、分配RDG资源
├─ TileClassification → 按材质分类Tile
├─ GenerateSamples → 随机选择光源、历史引导
└─ MarkVSMPages → 标记VSM需要渲染的页面
    ↓
（VSM渲染Shadow Maps）
    ↓
RenderMegaLights()
├─ RayTrace → Screen Traces + HW RT / VSM阴影 + BRDF着色
├─ Resolve → Upscale + 累积Diffuse/Specular
└─ DenoiseLighting → Spatial + Temporal降噪 → SceneColor
```

## 🆚 传统方案对比

| 维度 | MegaLights | 传统Tiled/Clustered |
|------|------------|---------------------|
| 光源上限 | 无实际上限 | ~256/Tile |
| 阴影 | 全部光源可投射 | 仅少量光源 |
| 开销曲线 | 近乎恒定 | 线性增长 |
| 噪点 | 有（需降噪） | 无 |
| 硬件要求 | RT或VSM支持 | 通用GPU |
