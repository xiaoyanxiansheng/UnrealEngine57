# 3.2 Lumen全局光照 - 速查卡

## 🎯 核心结论

1. **动态GI**：无需烘焙的全局光照，支持动态场景（昼夜循环、可破坏环境）
2. **Surface Cache**：场景表面用Mesh Cards（6朝向卡片）表示，缓存到Atlas纹理
3. **Screen Probe**：屏幕空间探针（16×16像素间距），Halton采样+时间滤波降噪
4. **Software RT**：Distance Field Sphere Tracing，不需要RTX硬件
5. **Radiance Cache**：3D空间Probe Grid，缓存远场间接光照

## 📊 Lumen三层架构

```
Surface Cache（场景表示层）
  Mesh Cards → Card Atlas（Albedo/Normal/Depth/Emissive）
  FLumenSceneData → FLumenCard → FLumenSurfaceCacheAllocator
      ↓
Ray Tracing（追踪层）
  Software RT：Global Distance Field + Sphere Tracing
  Hardware RT：DXR BVH加速（可选，需RTX）
      ↓
Screen Probe Gather（采样聚集层）
  Probe放置 → Probe追踪 → Temporal Filter → Final Gather
  Radiance Cache补充远场GI
      ↓
输出：Diffuse Indirect + Specular Indirect
```

## 🔗 核心入口函数

| 函数 | 文件 | 作用 |
|------|------|------|
| `UpdateLumenScene()` | `Engine/Source/Runtime/Renderer/Private/Lumen/LumenSceneRendering.cpp`，token：`FDeferredShadingSceneRenderer::UpdateLumenScene` | 每帧场景更新 |
| `RenderLumenSceneLighting()` | `Engine/Source/Runtime/Renderer/Private/Lumen/LumenSceneLighting.cpp`，token：`FDeferredShadingSceneRenderer::RenderLumenSceneLighting` | Surface Cache光照 |
| `ScreenProbeGather` | `LumenScreenProbeGather.cpp` | Screen Probe采集 |
| `TraceLumenRay` | `LumenSoftwareRayTracing.ush` | Software RT追踪 |

## 📊 性能数据（估算，RTX 4090 1080p）

| Pass | 估算耗时 | 占比 |
|------|---------|------|
| Surface Cache Update | ~2.5ms | 18% |
| Screen Probe Trace | ~3.5ms | 26% |
| Radiance Cache | ~1.5ms | 11% |
| Lumen Reflections | ~3.0ms | 22% |
| 其他（Filter/Gather/DF） | ~3.0ms | 22% |
| **总计** | **~13.5ms** | **100%** |

## 🆚 Unity对比

| 维度 | UE5 Lumen | Unity |
|------|-----------|-------|
| GI类型 | 完全动态 | Lightmap静态烘焙 |
| 追踪方式 | Distance Field SRT + DXR | SSGI（有限） |
| 反射 | 实时动态 | Reflection Probe（静态） |
| 动态物体GI | ✅ 完全支持 | Light Probe（低精度） |
| 烘焙时间 | 0（实时） | 分钟-小时 |
| 运行时开销 | ~13.5ms | ~0ms |
