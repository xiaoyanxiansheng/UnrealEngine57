# 2.5 RenderGraph - 速查卡

## 🎯 核心结论

1. **声明式**：描述资源依赖，自动管理屏障
2. **Transient资源**：帧内复用，减少显存
3. **自动调度**：根据依赖图确定执行顺序
4. **资源别名**：不同Pass复用同一显存
5. **Pass合并**：自动合并兼容Pass

## 📊 RenderGraph流程

```
声明阶段（Setup Phase）
├─ 创建资源（CreateTexture/Buffer）
├─ 添加Pass（AddPass）
└─ 声明依赖（Input/Output）
    ↓
编译阶段（Compile Phase）
├─ 资源生命周期分析
├─ 屏障插入
└─ 资源别名优化
    ↓
执行阶段（Execute Phase）
└─ 按依赖顺序执行Pass
```

## 🔗 核心API

```cpp
// 创建资源
FRDGTextureRef Texture = GraphBuilder.CreateTexture(...);

// 添加Pass
GraphBuilder.AddPass(
    RDG_EVENT_NAME("MyPass"),
    PassParameters,
    ERDGPassFlags::Raster,
    [](FRHICommandList& RHICmdList) { ... }
);
```
