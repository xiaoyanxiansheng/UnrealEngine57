# 2.1 数据流动机制 - 速查卡

## 🎯 核心结论

1. **双缓冲**：GameThread和RenderThread各持有独立数据副本
2. **FPrimitiveSceneProxy**：GT的UPrimitiveComponent在RT的代理
3. **SceneData同步**：GT修改 → 标记Dirty → RT帧末同步
4. **原子操作**：TAtomic实现无锁计数器
5. **内存对齐**：alignas(64)避免False Sharing

## 📊 数据流向

```
GameThread                 RenderThread
    │                           │
UPrimitiveComponent ──创建──> FPrimitiveSceneProxy
    │                           │
Transform修改 ──标记Dirty──> UpdateTransform()
    │                           │
材质修改 ──ENQUEUE──> UpdateMaterial()
```

## 🔗 核心类索引

| 类 | 职责 |
|-----|------|
| `FPrimitiveSceneProxy` | GT组件在RT的代理 |
| `FScene` | RT场景数据容器 |
| `FSceneView` | 单个视图的渲染数据 |
| `FViewInfo` | 视图的可见性信息 |

## 🔗 核心函数索引

| 函数 | 作用 |
|------|------|
| `AddPrimitive()` | 添加Primitive到场景 |
| `UpdatePrimitiveTransform()` | 更新Transform |
| `FScene::UpdateAllPrimitiveSceneInfos()` | 批量更新 |

## 🆚 Unity对比

| 维度 | UE5 | Unity |
|------|-----|-------|
| 代理模式 | SceneProxy | 无（直接访问） |
| 数据同步 | 显式ENQUEUE | 隐式同步 |
| 线程安全 | 手动管理 | C# GC |
