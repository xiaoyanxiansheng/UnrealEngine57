# 5.4 渲染性能优化 - 速查卡

## 🎯 核心结论

1. **先测量后优化**：`stat unit` → 判断瓶颈线程 → 再定位热点
2. **四层分析**：帧级 → Pass/函数级 → DrawCall/Shader级 → 硬件级
3. **CPU瓶颈**：通常是 DrawCall 过多 / InitViews 过慢
4. **GPU瓶颈**：通常是某个 Pass 过慢（Shadow / Lumen / BasePass）
5. **GPU-Driven**：Nanite + GPUScene 是消除 CPU 瓶颈的终极手段

## 📊 5分钟瓶颈定位流程

```
stat unit → 看 Game / Draw / GPU 哪个最长
  ├─ Draw 最长 → CPU瓶颈 → stat scenerendering → 看 InitViews / DrawCall
  ├─ GPU 最长  → GPU瓶颈 → stat gpu → 找最慢 Pass
  └─ Game 最长 → GameThread → 查 Blueprint / Physics / Tick
```

## 🔧 常用命令

| 命令 | 作用 |
|------|------|
| `stat unit` | 各线程耗时（判断瓶颈） |
| `stat gpu` | GPU 各 Pass 耗时 |
| `stat scenerendering` | Primitive / DrawCall / RT 时间 |
| `stat rhi` | DrawCall / Triangle / SetShaderParameter |
| `profilegpu` | 单帧 GPU 树状报告 |
| `stat nanite` | Nanite 统计 |
| `stat lumen` | Lumen 统计 |

## 📊 优化方法优先级

| 优先级 | 方法 | 效果 | 工作量 |
|--------|------|------|--------|
| **P0** | 开启 Nanite | 极高 | 极低 |
| **P0** | ISM / HISM | 极高 | 低 |
| **P0** | Merge Actors | 高 | 中 |
| **P1** | Shadow 降级 | 中-高 | 极低 |
| **P1** | Lumen 调整 | 中-高 | 极低 |
| **P1** | Shader 简化 | 中 | 中 |
| **P2** | Overdraw 治理 | 中 | 中 |

## 📊 性能目标（⚠️ 估算，1440p 60FPS）

| 指标 | 舒适 | 警戒 | 危险 |
|------|------|------|------|
| DrawCall | < 500 | 500-1500 | > 1500 |
| RenderThread | < 10ms | 10-15ms | > 15ms |
| GPU 总时间 | < 13ms | 13-16ms | > 16ms |
| Overdraw | < 2x | 2-5x | > 5x |

## 🔗 Scalability 排除法

```bash
sg.ShadowQuality 0              # 关阴影，看节省多少
sg.GlobalIlluminationQuality 0   # 关 GI，看节省多少
sg.PostProcessQuality 0          # 关后处理，看节省多少
sg.ReflectionQuality 0           # 关反射，看节省多少
```
