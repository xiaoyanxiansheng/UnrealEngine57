```
失败模式2：画面错误
├─ Nanite Mesh显示为黑色
├─ 墙壁漏光（Light Leak）
```

## 二、导入Nanite资产与验证

### 2.1 方法1：转换现有Mesh为Nanite（5分钟）

**场景**：你有一个高精度建筑模型（100万面）

```
1. 导入FBX模型
   Content Browser → Import → 选择 Building.fbx
   
2. 导入设置：
   ├─ Skeletal Mesh: False
   ├─ Import Normals: True
   ├─ Import Tangents: True
   └─ Combine Meshes: False

3. 转换为Nanite：
   右键 StaticMesh → Nanite → Enable Nanite Support
   
4. 验证Nanite状态：
   双击打开 StaticMesh Editor
   Details → Nanite Settings
   ├─ Enabled: ✅
   ├─ Position Precision: Auto
   └─ Percent Triangles: 100%（保留所有三角形）
```

**转换前后对比**：

| 属性 | 转换前 | Nanite | 说明 |
|------|--------|--------|------|
| **Triangle数** | 1,000,000 | 1,000,000 | 保持原始精度 |
| **LOD数量** | 5个（手动创建） | 自动（无限LOD） | Nanite自动生成Cluster层次 |
| **DrawCall** | 1个 | 合并到Nanite Pass | GPU-Driven批量处理 |
| **文件大小** | 50MB | 55MB | 轻微增加（Cluster元数据） |

### 2.2 方法2：使用Quixel Megascans（推荐，10分钟）

```
1. 打开Quixel Bridge
   Window → Quixel Bridge
   
2. 登录Epic账号（免费使用）

3. 搜索资产：
   - 类型：3D Assets → Architectural
   - 风格：Modern/Contemporary
   - 选择几个建筑Mesh

4. 下载并导入：
   选中资产 → Download → Add to Project
   
   自动导入：
   ├─ Nanite已启用 ✅
   ├─ LOD已优化 ✅
   └─ Material实例已配置 ✅
```

### 2.3 验证Nanite正常工作（关键步骤）

```
1. 在视口中放置Nanite Mesh

2. 查看Nanite统计：
   Console命令：stat nanite
   
   应该看到：
   ┌─────────────────────────────────┐
   │ Nanite Stats                    │
   ├─────────────────────────────────┤
   │ Visible Triangles: 1,234,567   │ ✅ 有数字
   │ Visible Clusters:  12,345      │ ✅ 有数字
   │ Instances:         100         │ ✅ 有数字
   │ Main Pass Time:    2.5ms       │ ✅ <5ms正常
   └─────────────────────────────────┘

3. 可视化Nanite Cluster：
   r.Nanite.Visualize.Overview 1
   
   看到彩色Cluster = Nanite工作正常 ✅
```

**常见导入问题**：

```
❌ Visible Triangles: 0
原因：Mesh未启用Nanite
解决：右键Mesh → Nanite → Enable

❌ 看到传统LOD切换（明显跳变）
原因：Nanite未生效，还在用离散LOD
解决：检查 r.Nanite.Enable 是否为1

❌ 黑色模型
原因：Normal/Tangent数据丢失或错误
解决：重新导入，勾选 Import Normals/Tangents
```

### 💡 检查点2

```
✅ stat nanite 显示 Visible Triangles > 0
✅ r.Nanite.Visualize.Overview 1 能看到彩色Cluster
✅ 没有黑面、闪烁等视觉问题
```

```
选中 Post Process Volume → Details

【Lumen Global Illumination】
├─ Method: Hardware Ray Tracing（RTX显卡）
├─ Lumen Scene Lighting Quality: 1.0（Epic）
├─ Final Gather Quality: 1.0（Epic）
├─ Scene Lighting Update Speed: 1.0（实时更新）
├─ Max Trace Distance: 10000（光线追踪距离，厘米）

【Lumen Reflections】
├─ Method: Hardware Ray Tracing
├─ Quality: 1.0（Epic）
├─ Ray Lighting Mode: Default
└─ Max Reflection Bounces: 1（多次反射，性能影响大）

【Exposure】（调试时关闭自动曝光）
├─ Metering Mode: Manual
├─ Exposure Compensation: 0
└─ Apply Physical Camera Exposure: False
```

**Quality Preset对比**：

| 预设 | Lumen Quality | GPU时间 | 画质 | 适用场景 |
|------|---------------|---------|------|---------|
| **Low** | 0.25 | ~2ms | 可接受 | 性能压力大 |
| **Medium** | 0.5 | ~4ms | 良好 | 中等画质 |
| **High** | 0.75 | ~6ms | 优秀 | 常规使用 |
| **Epic** | 1.0 | ~8ms | 极致 | 高端硬件 |
| **Cinematic** | 2.0 | ~15ms | 离线渲染级 | 截图/视频 |

```

步骤3：增加Radiance Cache采样
```

## 一、Nanite优化

### 1.1 精度调整（性能 vs 画质）

Nanite的"像素/边长"（Pixels Per Edge）决定LOD切换阈值：

```bash
# 默认值
r.Nanite.MaxPixelsPerEdge 1.0

# 降低精度（提升性能）
r.Nanite.MaxPixelsPerEdge 2.0  # 性能提升20-30%，画质轻微下降

# 提高精度（提升画质）
r.Nanite.MaxPixelsPerEdge 0.5  # 画质提升，性能下降30%
```

**实测对比**（100万面建筑）：

| 设置 | GPU时间 | Visible Triangles | 画质 |
|------|---------|-------------------|------|
| **0.5（高画质）** | 4.5ms | 2,000,000 | 极致 |
| **1.0（默认）** | 3.2ms | 1,000,000 | 优秀 |
| **2.0（性能）** | 2.3ms | 500,000 | 良好（2m外几乎无差异） |

**建议**：
- 高端显卡：0.5-1.0
- 中端显卡：1.0-1.5
- 预算紧张：2.0

# 第五部分：常见问题FAQ

> **核心目标**：提供快速排查路径，让你遇到问题时能在5分钟内定位根因

---

## 一、Nanite常见问题

### Q1：Mesh转Nanite后变黑

**原因**：Normal/Tangent数据丢失或编码错误

**解决**：

```
1. 重新导入FBX：
   Import Settings
   ├─ Import Normals: True ✅
   ├─ Import Tangents: True ✅
   └─ Recompute Normals: False

2. 或手动重新计算：
   StaticMesh Editor → Details
   └─ Build Settings → Recompute Normals: True
   
3. 重新启用Nanite：
   Nanite Settings → Enabled: ✅
```

### Q2：Nanite性能反而更差

**原因1**：Mesh太简单（<1万面）

```
Nanite有固定开销（Cluster构建、GPU Driven管线），
对简单Mesh这个开销可能超过收益。
解决：简单Mesh不用Nanite → StaticMesh → Nanite → Enabled: ❌
```

**原因2**：Instance数量少（<10个）

```
少量Instance的Nanite Overhead会抵消GPU-Driven批处理收益。
经验阈值：>50个Instance才考虑Nanite。
```

**原因3**：视锥外Culling失效

```
诊断：
stat nanite
查看 Culled Clusters 是否接近 0（应该远大于0）

解决：
r.Nanite.ViewMeshLODBias 0  # 确保未禁用Culling
```

### Q3：Nanite在某些角度闪烁

**原因**：Cluster边界精度问题或Z-Fighting

**解决**：

```
StaticMesh → Nanite Settings
├─ Position Precision: Auto → Float
└─ Percent Triangles: 100% → 95%（允许轻微简化，减少边界问题）
```

---

## 二、Lumen常见问题

### Q1：GI延迟明显（拖影）

**原因**：Temporal积累导致GI更新慢于相机移动

**解决**：

```bash
# 提高更新速度
r.Lumen.ScreenProbeGather.UpdateSpeed 2.0  # 默认1.0

# 或关闭Temporal Filter（实时但会出现噪点）
r.Lumen.ScreenProbeGather.TemporalFilter 0
```

### Q2：漏光（Light Leak）

**原因**：墙壁太薄或Mesh有缝隙，Lumen Ray穿透

**解决**：

```
方法1：加厚墙壁
├─ 墙壁厚度 >20cm

方法2：增加Surface Cache精度
r.LumenScene.SurfaceCache.MeshCardsPixelsPerDistance 20 → 50

方法3：检查Mesh闭合性
├─ 在建模软件中修复缝隙和孔洞
```

### Q3：某些物体没有GI

**原因**：未加入Lumen Scene

**解决**：

```
StaticMesh → Details → Lighting
├─ Affect Distance Field Lighting: ✅
└─ Emissive Light Source: ✅（如果是发光物体）

Material → Details
└─ Used with Nanite: ✅（如果是Nanite Mesh的Material）
```

### Q4：性能突然暴跌

**诊断**：

```bash
stat lumen

如果看到：
Surface Cache Update: 50ms ⚠️（正常<1ms）

原因：大量动态物体同时触发Surface Cache重新捕获

解决：
1. 将静态物体改为 Mobility: Static
2. 降低更新频率：
   r.LumenScene.SurfaceCache.UpdateFrequency 0.5
3. 检查是否有大量物体同时改变Transform
```
