# 4.1 MeshDrawCommand - 速查卡

## 核心结论

1. **Command Pattern**：将DrawCall封装为FMeshDrawCommand（POD对象），可缓存、可延迟执行
2. **Static Mesh缓存**：AddStaticMeshes时预构建Command，每帧直接读取，Setup开销归零
3. **Dynamic Instancing**：Hash分组 + 精确验证，可减少可合批场景的DrawCall，幅度需实测
4. **State Cache**：排序后连续提交，跳过重复PSO/Shader/VB设置，可减少重复RHI状态设置
5. **CRTP模板**：BuildMeshDrawCommands编译期确定Shader类型，消除虚函数开销

## MDC生命周期

```
Static Mesh（缓存）：
AddStaticMeshes() → BuildMeshDrawCommands() → 缓存到FCachedMeshDrawCommandInfo
    ↓ 每帧：
InitViews() → 读取缓存 → VisibleCommands → Sort → Dynamic Instancing → Submit

Dynamic Mesh（每帧）：
AddMeshBatch() → BuildMeshDrawCommands() → 生成FMeshDrawCommand → Submit
```
