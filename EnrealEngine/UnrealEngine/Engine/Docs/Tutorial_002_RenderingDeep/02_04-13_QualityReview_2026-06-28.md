## 分章复审

### 02 SceneProxy

**Verdict**: pass with upgrades。主线强，椅子贯穿案例能承载注册、移动、重建、删除；侧车报告当前，无阻断。

**03-level gap**: `PackedIndex` / `PersistentIndex`、Proxy 子类快照差异、static/dynamic relevance 分流仍偏“定义 + 表格”，案例密度弱于 03。

**High-value upgrades**:

1. 在 `FScene::Update()` 的 `PackedIndex / PersistentIndex` 后增加两个 primitive 的 add/remove/swap 微案例，说明 packed 下标会变、persistent 句柄稳定。
2. 在“同一条生命周期，不同的快照内容”补 Instanced / Skeletal / Nanite 的局部快照差异案例。
3. 在 Proxy 虚函数体系与 draw 收集输入之间补一个 static cached 主体 + dynamic 调试/选中部分的双路径案例。

### 04 RHI

**Verdict**: pass with upgrades。RHI 抽象墙、资源生命周期、transition、command list 回放都清楚；源码克制好。

**03-level gap**: PSO、shader binding、五个合同的局部案例仍可更具体。另发现共享说明中 04 篇幅数字口径已漂移，需改成不依赖行数的边界判例。

**High-value upgrades**:

1. 开篇路线图后加“五个合同”小表：后端、资源、状态、PSO-binding、回放分别接收什么、交出什么、坏了查什么。
2. 在 Shader Binding 小节用红色金属材质把 BaseColor texture、roughness scalar、sampler、uniform buffer / bindless handle 落到参数数组。
3. 在 PSO 小节补首帧 cache miss -> async compile -> command list wait -> 下次 cache hit 的微案例。
