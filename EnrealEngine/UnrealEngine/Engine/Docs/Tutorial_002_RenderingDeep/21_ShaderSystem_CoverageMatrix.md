## 残余风险汇总

1. Registry、map layout、compile predicates与cook coverage随模块/loading/build配置变化；必须沿三阶段和candidate/gate/job链重核。
2. VF type、instance data、stream component、declaration、input stream、VF resource binding必须持续分开。
3. Parameter metadata、RHI layout、compiled parameter map、binding records、runtime batched parameters不得合并为“反射/绑定”。
4. Type registered、binary cooked、map complete/RT visible、shader ref、MDC built、PSO ready、recorded、queue submitted、GPU consumed均是独立完成层。
