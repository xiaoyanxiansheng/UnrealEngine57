## 2. 实际主线与边界

实际主线是：

`measurement contract -> real timelines/critical path -> tool depth and observation cost -> cost owner/data -> cross-frame state -> falsifiable hypothesis/last-valid-state -> single-variable experiment -> product decision -> automation/completion`

本章不是“优化开关清单”。Visibility、MeshDrawCommand、GPUScene、BasePass、Lighting、VSM、Nanite、Lumen、PostProcess只承担成本owner与数据状态的定位入口；完整算法留在相应专题。CVar只承担实验控制，不能直接成为产品结论。

## 5. 残余风险

1. 双profile中的数字是教学示例，不是项目实测；实际验收必须生成自己的CSV、trace、ProfileGPU/capture、reference和平台数据。
2. ProfileGPU与深度capture有观测成本并可能改变async、parallel translate或residency时序；normal run与profile run必须分开。
3. 子系统符号会随UE实现重构，正文的稳定教学合同是owner/data/control/lifetime，而不是固定event字符串。
4. Dynamic resolution、streaming与temporal history会跨run污染baseline；只重置相机或CVar不足以恢复实验状态。

## 6. 独立 BODY 结论

**BODY PASS**。当前正文完整覆盖measurement/time/cost/CVar/platform/product completion：测量合同、真实时间线、工具成本、owner/data归因、跨帧状态、受控实验、双profile产品决策、自动化与完成门均闭环；冻结原版有价值的六步闭环和子系统导航已迁移并升级。本结论独立于旧sidecar、章节状态和公共Gate。
