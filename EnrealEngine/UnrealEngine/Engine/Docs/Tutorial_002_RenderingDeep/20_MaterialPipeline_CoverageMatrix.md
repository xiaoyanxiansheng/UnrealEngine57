## Worked Case / Last-Valid-State 覆盖

| 关联单元 | 正文案例 | 状态推进与调试证据 |
| --- | --- | --- |
| MAT-00、MAT-05 至 MAT-12 | 红色金属材质static switch | identity -> cache/layout/gates -> job/result -> finalized/complete -> GT/RT publication |
| MAT-03、MAT-13 | Roughness与texture runtime override | instance/proxy request -> cache/preshader -> UB/resource -> current draw binding，不触发无关compile |
| MAT-14 | BasePass default/fallback或missing | proxy fallback、coordinate missing、editor future request、PSO active skip分开 |
| MAT-15 | Nanite depth/coverage错与depth正常但material错 | 分别停在programmable raster与BasePass shading consumer |
| MAT-16 | legacy red-metal与Substrate slab对照 | translation output -> closure/storage requirements -> BasePass targets -> final writes |
| MAT-17 至 MAT-19 | editor/ODSC/cooked missing | 按环境和material重要性进入compile、future request、fallback或fatal |

## 残余风险汇总

1. Cached layout只表示候选coordinates；具体material gates、已有结果/in-flight work与job创建必须持续分层。
2. Nanite programmable raster与BasePass material shading不是同一consumer；Substrate translator output与render target/lighting contract也不是同一完成点。
3. Cooked missing的fallback/fatal结论依赖环境与material重要性，不能简化成单一路径。
4. `complete for material`、RT publication、shader ref、PSO ready、recorded、queue submitted、GPU consumed必须保持独立术语。
